# OctoAcme — Project Management Process Docs

This directory contains OctoAcme's project management process documents. The README below centralizes links to each process doc and provides a brief summary of the project management approach used across OctoAcme projects.

## Summary of OctoAcme project management processes

OctoAcme runs cross-functional projects with a focus on customer value, iterative delivery, and clear ownership. Each project has a named PM and Product Lead; teams operate with short, frequent feedback cycles (daily standups, regular planning, weekly syncs and demos at the end of sprints). Work is tracked on a project board (Backlog → Ready → In Progress → In Review → QA → Done), and PR/CI practices enforce small PRs, automated tests, and at least one approval before merging. Risks are captured in a Risk Register and escalated via a Level 1–3 path.

Use the linked documents below for detailed guidance on initiation, planning, execution, release, risk management, retrospectives, and roles.
